# 2025-오픈소스-SW-개발 기말 과제

이번 기말 과제는 팀 프로젝트 형태로 진행됩니다.
각 팀은 현업에서 실제 활용 중인 오픈소스 소프트웨어(Open Source Software, OSS)를 선정하여,
그 도입 배경, 기술적 아키텍처, 활용 사례, 성과 및 한계점 등을 심층적으로 분석합니다.

또한, 유사 사례 비교 분석 또는 간단한 프로토타입(Proof of Concept, PoC) 구현을 통해 해당 OSS의 실질적인 가치와 확장 가능성을 탐구합니다.

기말 과제 발표는 12월 12일(금)입니다.

프로젝트 수행 내용
조사 대상 선정

실제 산업 현장에서 활용 중인 OSS를 1개 선정합니다.

예시:

데이터베이스: MySQL, PostgreSQL, MongoDB

웹 프레임워크: Django, Spring Boot, React

DevOps 도구: Docker, Jenkins, Kubernetes, GitLab CI

모니터링 도구 : Prometheus, Grafana(시각화 등)
AI/ML 프레임워크: TensorFlow, PyTorch, Hugging Face Transformers

조사 및 분석 항목

도입 배경 및 필요성:
기업이나 조직이 해당 OSS를 채택하게 된 이유, 기존 시스템 대비 개선 포인트

아키텍처 분석:
구성 요소, 시스템 구조, 동작 원리, 주요 기술 스택 등을 시각적으로 정리 (다이어그램 포함 권장)

활용 및 성과:
해당 OSS가 실제 서비스나 업무 환경에서 어떻게 활용되고, 어떤 효과를 냈는지

유사 사례 비교:
유사 목적의 다른 OSS 또는 상용 솔루션과 기능·성능·커뮤니티 측면 비교

프로토타입 제작(선택 또는 가산점):
OSS를 직접 설치·활용하여 기본 기능을 재현하거나 간단한 데모를 제작

주제 선정시 활용 방안
실제 산업 동향과 기술 적용 사례를 이해하기 위해 다음 출처의 콘텐츠 활용을 권장합니다.

빅테크 기업의 공식 컨퍼런스 / 세미나 발표 영상

예: Google Cloud Next, AWS re:Invent, Microsoft Build, if kakao, deview Naver

공식 기술 블로그 및 개발자 포럼

예: Google Developers Blog, AWS Architecture Blog, Netflix Tech Blog, 카카오 기술 블로그, 네이버 기술 블로그 등

기술 커뮤니티 및 리서치 채널

예: The Linux Foundation, CNCF Blog, OpenAI Dev Blog, Hugging Face Blog

오픈소스 프로젝트 공식 GitHub Repository 및 Wiki 등)
